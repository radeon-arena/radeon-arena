# Radeon Arena

A community-driven **LLM performance leaderboard for AMD Radeon GPUs**. Real
recipes, multiple runtimes (vLLM / SGLang / llama.cpp / MLC-LLM), full
reproducibility — every result carries the exact serve command, quantization,
GPU and cluster topology used to produce it.

> **Disclaimer** — This is an **independent, community-built project** and is not
> affiliated with or endorsed by AMD. It is a from-scratch reimplementation of
> the architecture popularized by *Spark Arena* (NVIDIA DGX Spark), retargeted to
> the AMD Radeon / ROCm ecosystem. The bundled dataset is **synthetic seed data**
> generated by a deterministic, physically-grounded performance model for demo
> purposes — it is **not** real hardware measurement. Wire up Firebase and submit
> `llama-benchy` runs to populate it with real numbers.

## Stack

| Layer        | Choice                                              |
| ------------ | --------------------------------------------------- |
| Framework    | Next.js 14 (App Router, route handlers)             |
| Styling      | Tailwind CSS                                         |
| Database     | Firebase Firestore (optional)                       |
| Auth         | Firebase Auth (Google) — gates `/admin` & submissions |
| Hosting      | Firebase App Hosting (Cloud Run) — see `apphosting.yaml` |

The leaderboard is served from a periodically-regenerated **snapshot** rather
than querying Firestore on every request (mirrors the original design, saves
read quota and survives traffic spikes).

## Quick start (offline / seed mode)

No Firebase required — the app falls back to the bundled deterministic dataset:

```bash
pnpm install
pnpm dev
# open http://localhost:3000
```

## With a real Firebase backend

1. Create a Firebase project, enable Firestore and Google Auth.
2. Copy `.env.example` to `.env.local` and fill in the `NEXT_PUBLIC_FIREBASE_*`
   values plus admin credentials (`GOOGLE_APPLICATION_CREDENTIALS` or
   `FIREBASE_SERVICE_ACCOUNT`).
3. Seed Firestore and deploy security rules:

```bash
pnpm seed                       # writes /benchmarks documents
firebase deploy --only firestore:rules,firestore:indexes
```

## API

| Endpoint                              | Description                                  |
| ------------------------------------- | -------------------------------------------- |
| `GET /static/snapshot`                | Full per-test leaderboard aggregate          |
| `GET /static/carousel`                | Homepage highlight reel                       |
| `GET /api/leaderboard/users`          | Contributor ranking (no PII / emails exposed) |
| `GET /api/leaderboard/organizations`  | Model-publisher ranking                       |
| `GET /api/benchmarks/:id`             | Full submission detail incl. recipe + tests   |
| `GET /api/recipes/:id/raw`            | Download a recipe as YAML                      |
| `POST /api/generate-recipe`           | Structure a serve command (auth-gated)        |

## Scripts

```bash
pnpm dev        # local dev server
pnpm build      # production build
pnpm seed       # load seed data into Firestore (needs admin creds)
pnpm snapshot   # emit snapshot.sample.json + carousel.sample.json
```

## Project layout

```
src/
  app/
    page.tsx                 # homepage
    leaderboard/page.tsx     # leaderboard + 5 tabs
    admin/page.tsx           # moderation console (auth-gated)
    static/{snapshot,carousel}/route.ts
    api/{leaderboard,benchmarks,recipes,generate-recipe}/...
  components/                # Header, Footer, Carousel, leaderboard views
  lib/
    types.ts                 # domain model
    seedData.ts              # deterministic synthetic dataset
    aggregate.ts             # pure snapshot/carousel builders
    dataSource.ts            # Firestore-or-seed IO layer
    scoring.ts               # contributor/org score model
    metrics.ts               # compare-models metrics
scripts/                     # seed + snapshot generators
```

## Notes on data hygiene

Unlike some leaderboards, `GET /api/leaderboard/users` here **does not** return
contributor email addresses. Keep PII out of public aggregates.
